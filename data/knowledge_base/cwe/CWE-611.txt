CWE-611: Improper Restriction of XML External Entity Reference ('XXE')

Category: Input Validation, XML Processing
Likelihood of Exploit: Medium
OWASP Top 10: A05:2021 - Security Misconfiguration

Description:
The software processes an XML document that can contain XML entities with URIs that resolve to documents outside of the intended sphere of control, causing the product to embed incorrect documents into its output. XML External Entity (XXE) attacks exploit XML parsers that are configured to process external entity references. Attackers craft malicious XML input containing DOCTYPE declarations that reference external files or URLs.

Attack Vectors:
- File Disclosure: DOCTYPE referencing file:///etc/passwd or application configuration files
- SSRF via XXE: DOCTYPE referencing internal network resources (http://169.254.169.254/latest/meta-data/)
- Denial of Service (Billion Laughs): Exponentially expanding entities that exhaust memory
- Port Scanning: Using XXE to probe internal network services

Common Consequences:
- Confidentiality: Read local files, application secrets, or internal network resources
- SSRF: Access internal services behind firewall
- Denial of Service: Parser exhaustion through recursive entity expansion

Detection Signals in Code:
- XML parsers configured without disabling external entity processing
- DocumentBuilderFactory without setFeature("http://apache.org/xml/features/disallow-doctype-decl", true)
- SAXParser or XMLReader without secure processing features enabled
- User-supplied XML passed directly to XML parsers
- SOAP endpoints, SVG upload endpoints, XML configuration import features

Mitigations:
- Disable DOCTYPE declarations entirely: setFeature("http://apache.org/xml/features/disallow-doctype-decl", true)
- Disable external entities: setFeature("http://xml.org/sax/features/external-general-entities", false)
- Use less complex data formats (JSON) when XML is not strictly required
- Patch/upgrade all XML processing libraries regularly
- Implement input validation to reject DOCTYPE declarations in user-supplied XML
- Use JAXB or other high-level XML frameworks with secure defaults

Related CWEs: CWE-827 (Improper Control of Document Type Definition), CWE-918 (SSRF)
Common Platforms: Java, .NET, PHP, Python, Ruby
SonarQube Rules: java:S2755, xml:S2755
